# 同人誌レビューサイト「世の中に同人誌を広めたい」取扱説明書!!!

このプロジェクトは、Next.jsで構築された同人誌レビューサイトです。
データベースを使用せず、ファイル（Markdown/MDX）だけで簡単に管理・運用できるように設計されています。

## 使い始める準備

ご自身のパソコンでサイトを表示・編集するための準備です。

1. **コマンドプロンプト（ターミナル）を開く**
   このフォルダ（`同人誌ブログ`）を右クリックして「ターミナルで開く」などを選択してください。

2. **サイトを起動する**
   以下のコマンドを入力して Enter キーを押します。
   ```bash
   npm run dev
   ```

3. **ブラウザで確認する**
   起動すると `http://localhost:3000` というURLが表示されます。これブラウザに入力してアクセスしてください。

## レビュー記事の書き方

記事は `src/content` フォルダの中にファイルを作るだけで追加できます。

### 1. ファイルの作成
`src/content` フォルダの中に、新しいファイルを作成します。
ファイル名の末尾は必ず `.mdx` にしてください。（例: `my-favorite-book.mdx`）

### 2. 記事のフォーマット
ファイルの先頭には、必ず以下の「設定情報（Frontmatter）」を書いてください。

```md
---
title: "記事のタイトルをここに書く"
date: "2026-01-06"
rating: 5
thumbnail: "/images/sample-cover.jpg"
excerpt: "記事一覧に表示される短い説明文をここに書きます。"
---

# ここから本文です

ここにレビュー記事の本文を書いていきます。
普通のブログと同じように書けます。

## 見出し
- リスト1
- リスト2

**太字** や *斜体* も使えます。
```

> **ポイント**: `rating` は 1〜5 の数字で星の数を指定できます。

### 3. 画像の貼り方
画像を使いたい場合は、まず画像ファイルを `public/images` フォルダに入れてください（フォルダが無ければ作ってください）。
その後、記事の中で以下のように指定します。

```md
![画像の説明](/images/my-image.jpg)
```

## サイトのカスタマイズ

### サイト名の変更
`src/components/Header.tsx` ファイルを開き、以下の部分を書き換えてください。
```tsx
<Link href="/">
  ここに新しいサイト名
</Link>
```

### 年齢確認（R18）機能の設定
`src/components/AgeVerificationModal.tsx` で制御しています。
- 「いいえ」を押したときの飛び先を変えたい場合: `handleDeny` 関数内の URL を変更してください。

### 色やデザインの変更
- 全体の色: `src/app/globals.css`
- トップページのデザイン: `src/app/page.module.css`
- 記事の見た目: `src/app/blog/[slug]/page.module.css`

## フォルダ構成の説明

このプロジェクトの主なファイルは以下の場所にあります。

- **`src/content/`**: 大事なデータです。ここに全ての記事ファイル（.mdx）が入っています。
- **`src/components/`**: サイトの部品です。
    - `Header.tsx`: ページ上部のメニュー
    - `Footer.tsx`: ページ下部のコピーライト
    - `AgeVerificationModal.tsx`: 年齢確認画面
    - `ReviewCard.tsx`: 記事一覧のカード
- **`src/app/`**: ページの本体です。
    - `page.tsx`: トップページ
    - `blog/[slug]/page.tsx`: 記事詳細ページ
- **`public/`**: 画像などの静的ファイル置き場です。

## 無料で公開する方法

このサイトは [Vercel](https://vercel.com/) というサービスを使うと、完全に無料で公開できます。

1. GitHubにこのプロジェクトのコードをアップロードします。
2. Vercelに登録・ログインします。
3. "Add New Project" から GitHubのリポジトリを選んで "Deploy" するだけです。

設定は自動で認識されるため、難しい操作は不要です。

## よくある質問・トラブルシューティング

### Q. 記事が表示されません
- ファイル名が `.mdx` になっていますか？
- ファイルの先頭にある `---` で囲まれた設定情報は正しいですか？
- 再度 `npm run dev` でサーバーを再起動してみてください。

### Q. 画像が表示されません
- 画像は `public` フォルダに入っていますか？
- 記事内で指定するパスは `/public` を省略して `/images/...` のように書いてください。

### Q. 404エラーが出ます
- 開発サーバー（`npm run dev`）を再起動してみてください。
- 記事のファイル名（スラッグ）が重複していないか確認してください。
